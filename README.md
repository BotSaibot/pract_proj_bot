# Практический проект telegram-бот
## Техническое задание
Необходим специальный аккаунт в Telegram, для демонстрационных целей, интегрируемый с другими сервисами, автоматически обрабатывающий и отправляющий сообщения.

## Реализовано
* Модуль `bs4_based_parser`:
	* служебная функция `decoder_str_to_page()` &#8212; декодирует строку в страницу (int);
	* служебная функция `decoder_str_to_params()` &#8212; декодирует строку в параметры (dict);
	* служебная функция `get_general_info()` &#8212; получает URL&#8211;адрес хоста, общее количество результатов и общее количество страниц;
	* служебная функция `get_response()` &#8212; возвращает проанализированный HTML&#8211;документ ответа из источника по URL&#8211;адресу;
	* служебная функция `simply_traversal()` &#8212; простой разбор документов;
* Модуль `config`, содержащий токен для разработанного бота;
* Модуль `handlers`:
	* класс `FSMParser` группы состояний для машины состояний;
	* функция&#8211;обработчик `cancel_handler()` &#8212; отменяющая текущую команду, по команде пользователя;
	* служебная функция `continue_res()` &#8212; продолжает получать результат для анализатора;
	* функция&#8211;обработчик `debug_message_handler()` &#8212; показывает отладочное сообщение;
	* функция&#8211;обработчик `help_handler()` &#8212; показывает справочную информацию;
	* функция&#8211;обработчик `main_menu_handler()` &#8212; вызывает главное меню;
	* функция&#8211;обработчик `parser_go_to_page_handler()` &#8212; переходит на страницу результатов анализатора по номеру;
	* функция&#8211;обработчик `parser_handler()` &#8212; вызывает результаты анализатора;
	* функция&#8211;обработчик `parser_menu_handler()` &#8212; вызывает меню анализатора;
	* функция&#8211;обработчик `parser_nav_transition_failure()` &#8212; информирует об отказе и его причине;
	* функция&#8211;обработчик `parser_set_params()` &#8212; задает параметры анализатора;
	* служебная функция `process_cancel()` &#8212; сбрасывает состояние машины состояний;
	* функция&#8211;обработчик `process_set_params()` &#8212; обрабатывает сообщения пользователя с параметрами;
	* функция&#8211;обработчик `show_id_handler()` &#8212; показывает идентификатор пользователя;
	* функция&#8211;обработчик `start_handler()` &#8212; показывает приветственное сообщение;
	* функция&#8211;обработчик `stop_handler()` &#8212; останавливает бота;
* Модуль `kb`:
	* служебная функция `get_nav_parser_bottom_kb()` &#8212; предоставляет навигационную клавиатуру анализатора;
	* константа `main_menu_buttons` &#8212; содержит лист с объектами&#8211;кнопками главного меню;
	* константа `back_to_main_menu_button` &#8212; содержит объект&#8211;кнопку возврата в главное меню;
	* константа `back_to_parser_button` &#8212; содержит объект&#8211;кнопку возврата в анализатор;
	* константа `main_menu` &#8212; содержит объект&#8211;клавиатуру главного меню;
	* константа `exit_kb` &#8212; содержит объект&#8211;клавиатуру-ответ главного меню;
	* константа `iexit_kb` &#8212; содержит объект&#8211;клавиатуру возврата в главное меню;
	* константа `parser_menu_kb` &#8212; содержит объект&#8211;клавиатуру анализатора;
	* константа `parser_params_kb` &#8212; содержит объект&#8211;клавиатуру параметров анализатора;
* Модуль `main`, являющийся точкой входа;
* Модуль `params`:
	* служебная функция `convert_to_original_keys()` &#8212; преобразует ключи словарей к изначальному виду;
	* служебная функция `load_params()` &#8212; загружает параметры из файла&#8211;хранилища;
	* служебная функция `unload_params()` &#8212; выгружает параметры в файл&#8211;хранилище;
* Модуль `testcase` для тестирования средствами `unittest`:
	* класс `TestGetResponse` &#8212; для тестирования служебной функции `get_response()`;
	* класс `TestParams` &#8212; для тестирования служебных функций `load_params()` и `unload_params()`;
	* класс `TestParserDecoder` &#8212; для тестирования служебной функции `decoder_str_to_params()`;
	* класс `TestParserPageDecoder` &#8212; для тестирования служебной функции `decoder_str_to_page()`;
* Модуль `text`:
	* словарь&#8211;константа `buttons` &#8212; содержит тексты всех объектов&#8211;кнопок;
	* словарь&#8211;константа `layouts` &#8212; содержит тексты всех функций&#8211;обработчиков;
